Panasonic Let's Note <3 Linux

Musa Haydar | Nov 17, 2025

Ever since I saw the Linus Tech Tips video reviewing a Panasonic Let's Note, I've wanted to get my hands on one to use as a Linux machine. The 2022 model featured in the video was the last that included a DVD reader, one of the features that drew me to the laptop, so I knew I'd be picking up an older model.

The model I picked up was a $140 order from a Japanese reseller, a CF-SZ6 from 2016. This model features plenty of I/O (3x USB-A, SD Card, DVD, HDMI, Ethernet, and some other stuff for which I have no need), a 7th-gen Intel Core i5, and its iconic circular trackpad. It's not a particularly thin laptop, but has a lightweight metal chassis, with the laptop being smaller than and weighing less than my M2 MacBook Air.

The appeal of repurposing this old business laptop is similar to that of an older ThinkPad as a Linux machine (a laptop which has earned something of a cult following), especially given I don't intend to put it to any challenging graphics or computing tasks.

Running Linux

The Let's Note came pre-installed with a Windows 11 in Japanese, which ran rather slow (to no one's surprise). The English language pack was not pre-installed and although Windows 11 wasn't unusably slow on the device, it was annoying to deal with, so I moved on.

My distro of choice for this machine was Fedora 43, since the last time I had been using Linux, it was RHEL running on university research servers. I went with the MATE+Compiz spin, since as a kid I had (without knowing what Linux was at the time) used a machine running Fedora 18 or 19 (I'm not sure which) with GNOME 2. Then, in highschool, when I actually learned what Linux is and how to use it, I ran CentOS but was not pleased with GNOME 3 at all. After that I switched to Ubuntu and then Pop_OS. It's been a long time since GNOME 3, and I hear that later versions of GNOME have improved a lot, but for now I'm sticking with MATE (a fork of GNOME 2) for the nostalgia.

Not Let's Note specific, but unfortunately, the main-menu editor (Mozo) seems broken out of the box in the latest Fedora MATE+Compiz spin. Onformation on this issue is available at this thread about Slackware. If there are other such issues, that may give me a reason to try out the latest GNOME.

Circular Scrolling

Even in the pre-installed Windows, circular scrolling wasn't working out of the box. Likely the reseller never installed the proper Windows drivers. And neither did I.

To get circular scrolling to work on Linux, you have to use X11 with Synaptics drivers (instead of Wayland, which was running by default) since libinput does not support the feature. There's information about this on the Arch Linux wiki for a different model of the Let's Note but the idea is the same.

The Fedora MATE+Compiz spin I had installed seemed to have the drivers included, but in order to run X11 so that the trackpad could be used, I had to install SDDM:

sudo dnf install sddm-x11

Then, I created the a configuration file /etc/X11/xorg.conf.d/70-synaptics.conf similar to the Arch Linux wiki, though I removed the edge boundaries:

Section "InputClass"
    Identifier "touchpad catchall"
    Driver "synaptics"
    MatchIsTouchpad "on"
    MatchDevicePath "/dev/input/event*"
    Option "TapButton1" "1"
    Option "TapButton2" "3"
    Option "TapButton3" "2"
    Option "CircularScrolling" "on"
    Option "CircScrollTrigger" "0"
    Option "CircularPad" "on"
EndSection

After restarting the laptop, X11 was running and circular scrolling works.
